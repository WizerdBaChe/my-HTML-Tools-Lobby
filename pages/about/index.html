關於 HTML Tools Lobby

設計理念、使用方式與維護指引

專案簡介

HTML Tools Lobby 是一個以 GitHub Pages 為基礎的靜態工具集合站， 目標是把多個獨立的小工具整理到同一個靜態網站中， 降低工具分散、路徑混亂與維護成本。

大廳頁透過讀取 tools.json 自動產生工具卡片，支援搜尋、分類篩選、排序與精選展示，完全不需要後端服務。

設計原則

⚡

靜態優先

優先使用靜態 HTML / CSS / JavaScript，不依賴後端，不引入不必要的框架。

🧩

工具彼此獨立

每個工具位於 tools/<tool-id>/index.html，各自管理資源，避免互相污染。

📋

單一資料來源

tools.json 是工具卡片、精選區塊、排序、狀態標記的唯一資料來源。新增或調整工具只需更新此檔案。

🚀

MVP 優先

先做最小可用版，確認不破壞既有結構後，再考慮功能擴充與視覺升級。

新增工具流程

  • 建立工具資料夾：tools/<tool-id>/index.html
  • 在 tools.json 新增一筆工具資料
  • 填寫必要欄位：id、name、path、category、status
  • 從大廳確認卡片正常顯示、搜尋與預覽功能可用
  • 若有明顯功能變更，補到更新紀錄頁

工具狀態說明

  • READY功能穩定，可正常使用
  • BETA基本功能可用，仍在持續測試中
  • DRAFT開發中，尚未正式發布
